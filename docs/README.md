# Учет заявок на ремонт климатического оборудования (Учебная практика, неделя 2)

Приложение реализует модуль учета заявок:
- создание/редактирование заявок;
- назначение специалистов;
- комментарии специалистов по заявке;
- поиск и фильтрация заявок;
- расчет статистики: количество выполненных, среднее время ремонта, статистика по неисправностям.

## Стек
- Python 3.12
- FastAPI + Jinja2 (HTML интерфейс)
- SQLAlchemy 2.x
- PostgreSQL 16
- Docker Compose
- Pytest (минимальный функциональный тест)

## Запуск (Docker Compose)

1) Создать `.env`:
```bash
cp .env.example .env
```

2) Запуск:
```bash
docker-compose up --build
```

Что произойдёт:
- `db` создаст таблицы из `db/init.sql`
- `importer` загрузит данные из `data/import/*.csv`
- `app` поднимет веб-интерфейс на `http://localhost:8000`

## Вход в систему
Тестовые логины/пароли берутся из `data/import/inputDataUsers.csv`.
Например:
- Менеджер: `login1 / pass1`
- Специалист: `login2 / pass2`
- Оператор: `login4 / pass4`
- Заказчик: `login7 / pass7`

## Основные страницы
- `/ui/login` — вход
- `/ui/requests` — список заявок
- `/ui/requests/new` — новая заявка
- `/ui/requests/{id}` — карточка заявки + комментарии
- `/ui/statistics` — статистика (только Менеджер)
- `/ui/users` — пользователи (только Менеджер)
